tideWise 시스템 완전 이해 가이드
========================================================================================================

📈 **tideWise란?**
tideWise는 한국투자증권 KIS OpenAPI를 활용한 완전 자동화 주식 매매 시스템입니다.
프로그래밍 지식이 없어도 쉽게 사용할 수 있도록 설계되었습니다.

🎯 **핵심 목표**
• 사람의 감정을 배제한 객관적 매매
• 24시간 시장 감시 및 자동 매매 실행
• 체계적인 리스크 관리로 손실 최소화
• 일관된 매매 원칙 적용으로 수익 극대화

========================================================================================================
📊 **tideWise의 두 가지 매매 방식**
========================================================================================================

🔹 **1. 자동 매매 (Production Auto Trading)**
   • 실행 방법: python run.py 실행 후 메뉴에서 "1" 선택
   • 운영 시간: 장 시작(09:00)부터 장 마감(15:30)까지 자동 운영
   • 매매 간격: 5초마다 시장 모니터링
   • 특징: 중장기 관점의 안정적 매매

🔹 **2. 단타 매매 (Day Trading)**
   • 실행 방법: python run.py 실행 후 메뉴에서 "2" 선택  
   • 운영 시간: 장중 급등주 발견 즉시 매매
   • 매매 간격: 3분~1분 간격으로 빠른 매매
   • 특징: 당일 내 매수/매도 완료하는 초단타 매매

========================================================================================================
💰 **핵심 매매 원칙**
========================================================================================================

🛡️ **리스크 관리 (절대 원칙)**
   • 최대 포지션: 계좌 잔고의 20%까지만 투자
   • 손절 기준: -3% 도달 시 무조건 매도
   • 익절 기준: +5% 도달 시 이익 실현
   • 최대 동시 보유 종목: 5개 이하
   • 1일 최대 손실한도: 계좌 잔고의 5%

💹 **매수 기준 (엄격한 조건)**
   • RSI 85 이하 (과매수 구간 회피)
   • 단기 이동평균(5일) > 장기 이동평균(20일)
   • 거래량이 평상시 대비 1.05배 이상
   • 상승 모멘텀 0.3% 이상
   • 데이비드 폴 거래량 검증 통과

📉 **매도 기준 (신속한 판단)**
   • 손절: 매수가 대비 -3% 도달 시
   • 익절: 매수가 대비 +5% 도달 시  
   • 추세 반전: 기술적 지표 약화 시
   • 거래량 급감: 매수세 소멸 감지 시
   • 장마감 5분전: 보유 종목 강제 정리

========================================================================================================
⚡ **한국 시장 특화 기능**
========================================================================================================

🚨 **VI(변동성 중단) 대응**
   • 상승 VI 발생: 즉시 매수 또는 보유 (상승 기회 활용)
   • 하락 VI 발생: 즉시 시장가 전량 매도 (손실 방지)
   • VI 감지 시 최우선으로 처리 (다른 모든 조건보다 우선)

🕐 **시간 관리 시스템**
   • 장 시작 전: 전일 보유 종목 정리 및 시스템 준비
   • 장 운영 중: 실시간 모니터링 및 자동 매매
   • 14:30 이후: 신규 매수 금지
   • 14:50 이후: 모든 보유 종목 강제 매도

📈 **급등주 포착 시스템**
   • 실시간 급등주 스크리닝
   • 거래량 급증 종목 우선 분석
   • 작전주/허수 급등 필터링
   • 진정한 상승 종목만 매수 대상 선별

========================================================================================================
🔧 **시스템 설정 및 안전 장치**
========================================================================================================

⚙️ **기본 설정값**
   • 매매 실행: 활성화 (실제 거래)
   • 시뮬레이션 모드: 비활성화
   • 매매 사이클: 5초
   • 장마감 시간: 14:55 (5분 여유)
   • ESC 키 긴급 정지: 비활성화

🔐 **안전 장치**
   • API 연결 실패 시 자동 재연결 (최대 3회)
   • 연속 실패 5회 시 30분 대기 후 재시작
   • 메모리 사용량 90% 초과 시 경고
   • 디스크 용량 95% 초과 시 정지
   • 텔레그램 실시간 알림 발송

📱 **모니터링 기능**
   • 실시간 포트폴리오 현황
   • 매매 내역 상세 로그
   • 수익률 통계 및 분석
   • 시스템 헬스 체크
   • 오류 발생 시 즉시 알림

🚀 **시작하기 (초보자 가이드)**
========================================================================================================

1️⃣ **설치 및 설정**
   • python auto_setup.py 실행하여 자동 설치
   • Policy/Register_Key/Register_Key.md에 API 키 입력
   • 텔레그램 봇 토큰 설정 (선택사항)

2️⃣ **첫 실행**
   • python run.py 실행
   • 메뉴에서 원하는 매매 방식 선택
   • 1번: 안정적 자동매매 (추천)
   • 2번: 적극적 단타매매 (고급자용)

3️⃣ **모의투자로 연습**
   • Register_Key.md에서 MOCK 계정 설정
   • 실제 돈 없이 시스템 작동 확인
   • 충분히 익숙해진 후 실계좌 전환

4️⃣ **실전 투자**
   • Register_Key.md에서 REAL 계정 설정
   • 소액부터 시작하여 점진적 확대
   • 매일 결과 분석 및 전략 개선

⚠️ **주의사항 및 면책**
========================================================================================================

🚨 **투자 위험 안내**
   • 주식 투자는 원금 손실 위험이 있습니다
   • 과거 수익률이 미래 수익을 보장하지 않습니다
   • 본 시스템은 투자 보조 도구일 뿐, 수익을 보장하지 않습니다
   • 투자 결정과 그 결과에 대한 책임은 사용자에게 있습니다

🛑 **시스템 한계**
   • 예측 불가능한 시장 급변 시 대응 한계
   • 네트워크 장애 시 매매 누락 가능성
   • API 서버 점검 시 일시적 중단
   • 극단적 시장 상황에서는 수동 개입 필요

✅ **권장사항**
   • 투자 금액을 여유자금 범위 내로 제한
   • 정기적인 수익률 점검 및 전략 조정
   • 시장 상황 변화에 따른 설정값 최적화  
   • 다양한 투자 방법과의 분산 투자
